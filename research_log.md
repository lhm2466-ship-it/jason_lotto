# 리서치 및 벤치마킹 기록 (Research Log)

이 문서는 외부 로또 관련 서비스 분석 내용과 우리 **JASON LOTTO** 프로젝트의 차별점 및 발전 방향을 기록합니다.

---

## 1. 외부 사이트 분석 (Reference Site)
*   **URL:** `https://parkminkyu.github.io/newLotto/lottoWin.html`
*   **주요 특징:** 
    *   **초고속 시뮬레이션:** 사용자가 선택한 번호로 1등 당첨될 때까지 무한 반복 추출하여 확률을 체감하게 함.
    *   **데이터 시각화:** 실시간 자산 변화, 투자 대비 수익률(ROI), 번호별 출현 빈도 그래프 제공.
    *   **경제적 지표:** 로또 구매 비용과 적금 수익을 비교하여 기회비용 강조.
    *   **외부 서비스:** Disqus(디스커스) 댓글 시스템 활용.

| 항목 | 외부 사이트 (시뮬레이터) | JASON LOTTO (당첨 예측 도구) |

| **핵심 목적** | 로또 확률의 희박함 증명 (시뮬레이션) | 가중치 기반 당첨 확률 극대화 (번호 생성) |

## 3. 전략적 인사이트 (Strategic Insights)

### ✅ 유지 및 강화해야 할 강점 (Unique Selling Points)
1.  **독창적인 가중치 알고리즘:** 단순 랜덤이 아닌 데이터 기반의 '과학적 생성' 이미지를 지속적으로 강조.
2.  **자체 Firebase 댓글 시스템:** 데이터 소유권을 유지하며 실시간 소통이 가능하다는 점은 커뮤니티 활성화에 매우 유리함.
3.  **완전 자동화 운영:** 운영자의 개입 없이 매주 최신 데이터가 반영되는 기술적 우위 유지.

### 🚀 향후 도입 고려 기능 (Inspired by Research)
1.  **가중치 유효성 시뮬레이터:** "우리 가중치 번호가 일반 랜덤보다 얼마나 더 효율적인가"를 증명하는 간단한 비교 기능.
2.  **당첨 통계 시각화:** 최근 10회차 이내 가장 많이 나온 번호(Hot)와 적게 나온 번호(Cold)를 차트로 제공.
3.  **공유 기능 고도화:** 생성된 번호뿐만 아니라, "이번 주 당첨 확률 가중치 현황" 같은 정보성 이미지 공유 기능.

## 4. 보안 및 법적 주의사항 (Security & Copyright)
*   **코드 복제 금지:** 타 사이트의 소스 코드를 그대로 가져다 쓰는 것은 저작권 위반 및 검색 엔진 패널티(SEO 감점)의 위험이 큼.
*   **독자적 구현:** 타 사이트의 '아이디어'만 참고하되, 모든 기능 구현은 **Chart.js**나 **Firebase** 등 검증된 라이브러리를 활용해 우리만의 고유한 코드로 작성함.

## 5. 트러블슈팅: 토요일 밤 자동 업데이트 지연 및 수동 전환 (Troubleshooting)

*   **현상:** 매주 토요일 저녁 8시 45분 이후 최신 당첨 번호가 즉시 업데이트되지 않는 문제 발생.
*   **원인:** 
    *   동행복권(dhlottery.co.kr) 사이트가 토요일 밤 접속자 폭주 시 '간소화 페이지'로 전환되어 API 접근이 차단됨.
*   **해결책 및 변경 사항:**
    1.  **수동 업데이트 전환:** 자동 스케줄(cron)에 의존하는 대신, 사용자가 필요할 때 GitHub Actions에서 직접 실행하는 **수동 업데이트 방식**으로 변경함.
    2.  **update_lotto.js 개선:** 간소화 페이지 감지 시 명확한 에러 로그를 남기도록 하여, 수동 실행 시 현재 업데이트 가능 여부를 쉽게 파악할 수 있도록 함.
    3.  **실행 방법:** GitHub 저장소의 `Actions` 탭에서 `Lotto Data Manual Update` 워크플로우를 선택하고 `Run workflow` 버튼을 눌러 실행.
